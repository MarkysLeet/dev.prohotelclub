# Инструкции для ИИ (AI Agent Instructions)

Данные правила действуют ВСЕГДА при любом изменении кода в этом репозитории, независимо от того, как была поставлена задача. В данном репозитории мы работаем в контексте проекта "ProHotelClub".

## 1. Использование MCP Linear
Любое логически завершенное изменение архитектуры, логики или UI должно фиксироваться в Linear. Запрещено создавать локальные файлы отчетов.

## 2. Языковой барьер (Strict Russian for Linear)
Твои внутренние рассуждения и логи в чате IDE могут быть на английском. НО все задачи, комментарии и отчеты, которые ты отправляешь в Linear, ДОЛЖНЫ БЫТЬ СТРОГО НА РУССКОМ ЯЗЫКЕ.

## 3. Логика отчетности (Не спамить)
Все новые тикеты и задачи должны быть связаны с проектом "ProHotelClub".
- **Если задача уже есть в Linear:** Переведи её в статус Done и оставь комментарий с отчетом.
- **Если задача поставлена напрямую в чате (ее нет в Linear):** После выполнения кода создай новую задачу сразу в статусе Done (как лог проделанной работы).
- **Работа с Milestones:** Рекомендуется выполнять крупные вехи (milestones) последовательно, задача за задачей, фиксируя каждую часть согласно этим правилам.

## 4. Структура отчета (Обязательный формат)
В отчетах не нужно указывать названия веток, коммитов или списки измененных файлов. Каждый твой комментарий или созданный тикет должен строго содержать следующие 3 пункта:
- **Что сделано:** (Краткое техническое описание)
- **Зачем это сделано:** (Какую ошибку или проблему это решает)
- **Ценность для проекта:** (Как это улучшает конверсию, премиальность UX или скорость загрузки)

## 5. Предложения по улучшению (Proactive Suggestions)
Если во время работы у тебя возникают идеи по улучшению чего-либо (кода, UI/UX, производительности), ты должен создать новую задачу (issue) в Linear. Опиши свою идею там (на русском языке), чтобы пользователь мог сам решить, стоит ли ее добавлять в проект. Самостоятельно, без подтверждения, такие изменения не внедрять.

## 6. Workflow Guideline (Autonomous Linear)
Для автономного забора задач используй `python3 scripts/linear_tool.py get_next_task`.
Для завершения: `python3 scripts/linear_tool.py complete_task <id> "<отчет>"`.
Все отчеты в Linear строго на русском (Что сделано / Зачем / Ценность).
Задачи выполняются согласно приоритетам: сначала задачи с приоритетом Urgent/High, затем Todo с учетом Milestones.
При получении задачи скрипт автоматически переводит ее в "In Progress".
